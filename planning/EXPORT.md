# Экспорт, схема и режим укладки

## Сохранение проекта

### Локальное автосохранение

Всё рабочее состояние (настройки комнаты, фото, кэш пикселизации,
история undo/redo без фото) автоматически сохраняется в IndexedDB
браузера. При закрытии и открытии вкладки проект восстанавливается
полностью.

IndexedDB — встроенное в браузер хранилище, не требует сервера
и внешних сервисов.

### Undo/redo — снимки без фото, лимит 50

История изменений хранит **снимки состояния всех настроек**
(комната, плитка, маски, параметры палитры, кадрирование), но
**не сами фотографии**. Замена фото в undo/redo не откатывается —
пользователь загружает фото заново при необходимости.

Лимит истории — 50 шагов. Снимки без фото — копейки по памяти.

Кнопки undo/redo доступны в основном UI (мобильное приложение,
Cmd+Z недоступен).

### Экспорт/импорт файла

**Формат:** один JSON-файл. Фотографии включены внутрь как base64.

На верхнем уровне JSON содержит поля:
- `format: "drape-project"` — сигнатура для распознавания
- `version: "1.0"` — версия формата для будущих миграций

Операции:
- **«Сохранить проект»** → скачивается JSON со всем содержимым.
- **«Открыть проект»** → загружается JSON, всё восстанавливается.

### Валидация при импорте

Три уровня проверки:
1. **Парсинг.** Если файл не парсится как JSON — ошибка
   «файл повреждён».
2. **Сигнатура.** Если нет поля `format: "drape-project"` —
   «это не файл проекта Drape».
3. **Версия.** Если `version` не совпадает с поддерживаемой —
   отказ «файл из другой версии». В будущем — миграции.
4. **Структура.** Если версия совпадает, но логика битая
   (например, маска ссылается на несуществующую стену) —
   загружается, что можно, остальное игнорируется с предупреждением.

Размер файла обычно 5–50 МБ в зависимости от количества и размера
фото. Это приемлемо для редкой операции бэкапа или передачи между
устройствами.

## Раздел «Схема и спецификация»

Отдельный раздел в приложении. Ничего не нужно скачивать для
просмотра — всё внутри.

### Компоновка экрана

Split-screen: экран делится на две части. Граница перетаскивается
вверх/вниз.

**Верхняя часть — развёртка:**
- Стены разложены в плоскость горизонтально
- Каждая стена — прямоугольник с сеткой плиток
- Маски-препятствия видны как пустые прямоугольники
- Между стенами зазор, в зазоре — название стены
- В каждой клетке: заливка цветом + номер цвета
- Зумится и двигается пальцами независимо от нижней части

**Нижняя часть — легенда и спецификация (прокручиваемая):**
- В шапке: размер плитки, ширина шва, цвет шва, итоговые размеры стен
- Поле **«Запас, %»** (по умолчанию 10%, можно поставить 0)
- Легенда: номер → квадратик цвета → hex-код
- Спецификация: для каждого цвета:
  - Итого штук
  - Колонка «с запасом» (если запас > 0)
  - Разбивка по стенам
  
  Пример: 🟦 #3A7FD5 — всего 320 шт. → с запасом (10%): 352 шт.
  → Стена 1: 180 / Стена 2: 140
- Кнопка «Скачать SVG» — открывает диалог настроек экспорта
- Кнопка «Режим укладки» — переход в пошаговый режим

## Экспорт SVG

Один файл содержит развёртку всех стен с сеткой плиток, легенду
и спецификацию. SVG масштабируется бесконечно, открывается в браузере
без программ, весит мало.

При большом количестве плиток (50 000+) для оптимизации размера
SVG используется механизм `<symbol>` / `<use>`: каждый цвет
определяется один раз и переиспользуется во всех плитках.

### Настройки экспорта SVG

Перед скачиванием — небольшой диалог:

**Масштаб:**
- **На экран** (по умолчанию) — оптимизировано под просмотр в браузере
- **Реальный размер 1:1** — атрибуты `width`/`height` задаются в мм
  с реальными размерами стен. Подходит для печати на плоттере как
  шаблона для бумажной мозаики или мелких плиток. Печать на A4/A3 —
  через драйвер принтера, приложение не делит на листы.

**Рисовать швы:**
- **Включено** (по умолчанию, если grout_width > 0) — между
  плитками виден шов цвета grout_color
- **Выключено** — плитки рисуются вплотную, как простая сетка
  (для бумажной мозаики без затирки, чтобы не возиться с
  вырезанием швов)

При grout_width = 0 переключатель неактивен.

## Режим укладки

Отдельный экран для пошаговой реализации мозаики (укладка плиток,
наклейка бумажек, покраска).

### Интерфейс

- **Большой блок текущей плитки**: цвет, номер цвета, координаты
  («Стена 1, ряд 5 от пола, плитка 12 слева»), размер плитки в мм.
- **Мини-превью стены**: схема стены с подсветкой текущей плитки
  и нескольких соседних. Тап на любую плитку — переход к ней.
- **Кнопки навигации**: «Предыдущая», «Следующая», «К плитке…».
- **Счётчик прогресса**: «247 из 1820».

Маски-препятствия пропускаются автоматически. Счётчик считает
только реально нужные плитки.

### Координаты плиток

Сквозная нумерация по сетке стены без учёта масок (плитка 12 — это
всегда 12-я слева от края, независимо от вырезов). Совпадает
с физической раскладкой.

### Режимы обхода

- **По рядам** (по умолчанию) — слева направо, снизу вверх. Удобно
  для физической плитки и кирпича — реальный порядок укладки.
- **По цветам** — все плитки одного цвета подряд. Удобно для бумажных
  пиксель-мозаик и раскраски: пользователь работает с одним цветом
  до конца, потом берёт следующий.

### Переключение между стенами

В режиме «по рядам» стены идут последовательно: после последней плитки
одной стены — переход на следующую. В режиме «по цветам» внутри одного
цвета сначала все плитки стены 1, потом стены 2, и т. д.

## UI настройки углов

Переопределение направления перекрытия объёмных плиток на углах
живёт в **форме настроек комнаты**, под списком стен. Каждый угол —
отдельная строка с селектором:

- «Автоматически» (по дефолтному правилу: толще выигрывает)
- «Стена N перекрывает»
- «Стена M перекрывает»

Рядом — мини-схема результата (две полоски с разной толщиной),
чтобы пользователь видел эффект до перехода в 3D.

При равной толщине плиток у соседних стен — дефолт «стена с меньшим
номером перекрывает», переопределяется так же.
