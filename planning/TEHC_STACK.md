# Технологический стек

## Фреймворк

**React.** Основной UI-фреймворк. Выбран за экосистему вокруг 3D
(React Three Fiber) и хорошую поддержку в Claude Code.

## 3D

**React Three Fiber (R3F) + Three.js.** R3F — обёртка над Three.js,
позволяет описывать 3D-сцену как React-компоненты.

Дополнительные пакеты:
- `@react-three/drei` — готовые OrbitControls, камеры, освещение
- Three.js — нативный доступ для InstancedMesh

**Instanced rendering обязателен.** Объёмные плитки рендерятся
через `InstancedMesh` (один меш на все плитки стены, разный цвет
на инстанс). Закладывается с этапа 5 как требование, не как поздняя
оптимизация.

С учётом лимитов 25 000 / 75 000 плиток на стену InstancedMesh
комфортно тянет на типичных мобильных устройствах.

## Обработка изображений

**Свой Web Worker (чистый JS).** Алгоритм area averaging и квантизация
(медианное сечение) написаны вручную — нужен доступ к цвету каждой
плитки для спецификации, готовые библиотеки это не закрывают чисто.

Выполняется в Web Worker + OffscreenCanvas.

**Совместимость браузеров:**
- **Chrome, Firefox, Edge, Yandex (на Chromium)** — OffscreenCanvas
  поддерживается давно, работает в Web Worker без проблем.
- **Safari ≥ 16.4** — поддерживает OffscreenCanvas.
- **Safari < 16.4 (старые iPhone)** — фоллбек на обычный canvas
  в главном потоке. UI на 1–3 секунды замораживается во время
  обработки, но работает. При запуске показывается одноразовое
  уведомление с предложением обновить браузер.

Передача данных между Worker и главным потоком — через
**Transferable Objects** (большие массивы передаются по ссылке,
не копируются).

## PWA — service worker через vite-plugin-pwa

Полноценный PWA: манифест + service worker. Стратегия — **precache
всех ассетов сборки**. После первой загрузки приложение работает
офлайн полностью (все проекты в IndexedDB).

Реализация — **vite-plugin-pwa**, генерирует service worker
автоматически.

**Стратегия обновления при выходе новой версии** — открытый вопрос
(см. OPEN_QUESTIONS Q16). Варианты: молча в фоне, тост «доступно
обновление», принудительная перезагрузка.

## Хранение данных

**IndexedDB для всего рабочего состояния.**

IndexedDB — встроенное в браузер хранилище, не требует сервера и
внешних сервисов. Отличия от localStorage:
- Лимит — десятки/сотни МБ (вместо 5 МБ)
- Хранит бинарные данные (фото) напрямую
- Чуть сложнее в коде, упрощается тонкой обёрткой `idb` (~5 КБ)

Хранится:
- Настройки комнаты, плитки, масок
- Фото (отдельное хранилище, ссылки по id)
- Кэш результата пикселизации
- История undo/redo (снимки настроек, **без фото**, ссылки на
  фото по id)

При каждом изменении состояния — автозапись в IndexedDB.

## Этапы реализации

### Этап 1 — Каркас
React-приложение, PWA-манифест + service worker (vite-plugin-pwa).
R3F. 3D-куб крутится в браузере. Автосохранение в IndexedDB.
Каркас undo/redo (снимки настроек).

### Этап 2 — Модель комнаты
Форма ввода стен (см) и плитки (мм). Маски на стене (только
препятствие). Настройка перекрытия в углах. Расчёт сетки с учётом
масок (округление в большую сторону). Лимиты 25 000 / 75 000 плиток
на стену. Только логика и числа.

### Этап 3 — 3D-вьюер (базовый)
Комната из стен в 3D, серые стены без мозаики. Орбитальная камера,
переключатель проекции (перспектива/изометрия). Flat-освещение.
Маски как пустые области. Полноэкранный отдельный раздел.

### Этап 4 — Пикселизация
Web Worker с area averaging. Загрузка фото, назначение на стены.
Плоское 2D-превью. 5 слайдеров цветокоррекции с дебаунсом.
Обязательная квантизация (одна палитра на проект, дефолт 32).
Кадрирование drag/pinch по развёртке с переключателем зазоров.
Палитра в боттом-панели (read-only), сортировка по
многочисленности/hue.

### Этап 5 — Мозаика в 3D
Результат пикселизации на стены в вьюере. Объёмные плитки через
InstancedMesh при толщине > 0. Цвет и ширина шва (включая для
плоских плиток).

### Этап 6 — Экспорт
Раздел схемы: split-screen, развёртка, спецификация. Поле «запас, %».
Кнопка «Скачать SVG» с настройками (масштаб 1:1, переключатель швов).
Экспорт/импорт JSON с валидацией версии.

### Этап 7 — Режим укладки
Текущая плитка крупно, мини-превью стены, два режима обхода
(по рядам / по цветам).

### Этап 8 — Полировка
Состояния загрузки и ошибок. Мобильные. Доводка UI. Фоллбек для
Safari < 16.4. Тестирование на реальных сценариях.
