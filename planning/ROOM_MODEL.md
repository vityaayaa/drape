# Модель помещения и плиток

## Единицы измерения

- Размеры стен и масок — в **сантиметрах** (см)
- Размеры плиток и швов — в **миллиметрах** (мм)
- Поля интерфейса всегда подписаны единицами

## Комната

Комната — набор стен, соединённых в замкнутый контур.
Конец одной стены = начало следующей, последняя замыкается на первую.
Углы между стенами: 90° в первой версии. Поле `angle` заложено
в модели для произвольных углов в будущем.

Понятия «толщина стены» в модели нет. Стены — это внутренние
поверхности, рисуемые в 3D как тонкие плоскости. В формуле сетки
участвует только толщина плитки соседних стен (`tile_thickness`),
которая физически выпирает в углу.

### Нумерация стен и обход контура

Стены нумеруются в порядке их добавления пользователем
(Стена 1, Стена 2, …, Стена N). Контур обходится в том же
порядке и замыкается: после Стены N — возврат к Стене 1.

Углы нумеруются по тому же контуру: Угол 1 — между Стеной 1
и Стеной 2, Угол 2 — между Стеной 2 и Стеной 3, …, Угол N —
между Стеной N и Стеной 1.

У каждой стены есть «левый сосед» (предыдущая стена в обходе)
и «правый сосед» (следующая стена). Это используется в формуле
сеточной ширины и в правиле перекрытия в углах.

## Параметры стены

| Поле | Единица | Описание |
|---|---|---|
| `length` | см | Длина стены |
| `height` | см | Высота стены |
| `wall_active` | bool | Стена существует (если выкл — стены нет) |
| `mosaic_active` | bool | Мозаика на стене (стена видна, мозаики нет) |
| `angle` | град | Угол к следующей стене (по умолчанию 90°) |
| `tile_overrides` | объект | Локальные переопределения параметров плитки (опц.) |

### tile_overrides — что можно переопределять

Через `tile_overrides` на уровне стены переопределяются любые
параметры плитки: `tile_width`, `tile_height`, `tile_thickness`,
`grout_width`, `grout_color`. Поле, не указанное в overrides,
берётся из глобальных параметров проекта.

## Маски (объекты на стене)

Маска — прямоугольная область на стене, которая не покрывается
плиткой. Используется для лестниц, дверей, окон, шкафов, розеток
и любых других объектов, перекрывающих часть стены.

В v1 маска имеет только один режим — препятствие. Декоративные
маски убраны.

| Поле | Единица | Описание |
|---|---|---|
| `name` | строка | Имя маски («лестница», «дверь») |
| `wall_id` | id | На какой стене расположена |
| `x` | см | Координата левого нижнего угла от левого края стены |
| `y` | см | Координата левого нижнего угла от пола |
| `width` | см | Ширина маски |
| `height` | см | Высота маски |

### Учёт плиток в маске

Плитка вырезается из спецификации **тогда и только тогда, когда
она полностью находится внутри маски**. Частично попадающая плитка
считается целой и идёт в спецификацию.

Физическая обрезка плитки по контуру маски (если нужно) — задача
пользователя при укладке. Альтернативный подход (вырезать частичные
плитки тоже) оставлял бы пустое пространство по контуру маски,
что нелогично для всех сценариев (бумага, краска, плитка) — проще
купить с запасом и обрезать на месте.

### Сложные формы

Описываются набором прямоугольных масок. П-образная лестница на
одной стене — 2–3 прямоугольных маски, аппроксимирующие профиль.
Для большинства реальных сценариев достаточно одной большой
прямоугольной маски по габариту объекта целиком.

На одной стене может быть любое количество масок.

### Маски и распределение фото-панорамы

Маски не участвуют в распределении фото между стенами. Фото
растягивается по сеточной ширине стен (см. ниже). Часть фото,
попавшая на маску, просто не отрисовывается. Это решение
зафиксировано в DECISIONS №66.

## Параметры плитки

Задаются глобально для всего проекта. На уровне стены можно
переопределить через `tile_overrides`.

| Поле | Единица | Описание |
|---|---|---|
| `tile_width` | мм | Ширина плитки |
| `tile_height` | мм | Высота плитки |
| `tile_thickness` | мм | Толщина плитки (0 = плоская, >0 = объёмная) |
| `grout_width` | мм | Ширина шва (по умолчанию 0) |
| `grout_color` | hex | Цвет шва (используется при grout_width > 0) |

В v1 плитка всегда четырёхугольная (квадрат или прямоугольник
по соотношению `tile_width` × `tile_height`). Поле `shape`
из модели убрано как зомби-поле (см. DECISIONS №67).

Толщина плитки и ширина шва — независимые параметры. Можно сделать
плоские плитки (толщина 0) с зазором между ними (шов > 0) — для
бумажной мозаики или раскраски с разделительными линиями.

## Углы и перекрытие

На стыке двух стен с объёмными плитками (`tile_thickness > 0`
у обеих) одна стена перекрывает другую. Дефолтное правило:

1. **Толще перекрывает тоньше** — стена с более толстой плиткой
   перекрывает соседнюю с более тонкой.
2. **При равной толщине** — стена с меньшим порядковым номером
   перекрывает соседнюю.
3. **Пользователь может переопределить** для каждого угла вручную
   (см. UI настройки углов в EXPORT).

Если у одной из соседних стен `mosaic_active = false` или
`tile_thickness = 0`, перекрытия в этом углу нет — соответствующее
вычитание в формуле сеточной ширины не применяется.

## Расчёт сетки плиток (сеточная ширина)

Для каждой активной стены с активной мозаикой:

```
сеточная_ширина = length 
  − tile_thickness_левого_соседа  (если он перекрывает в углу слева)
  − tile_thickness_правого_соседа (если он перекрывает в углу справа)

сеточная_высота = height

колонки = floor((сеточная_ширина + grout_width) / (tile_width + grout_width))
строки  = floor((сеточная_высота + grout_width) / (tile_height + grout_width))
```

Формула учитывает, что между N плитками N−1 шов, а не N.

При `tile_thickness = 0` у соседа вычитание даёт ноль —
формула возвращает полную длину стены, что и ожидается для
плоских плиток.

### Нумерация плиток на стене

Ряды плиток нумеруются от пола вверх: ряд 1 — нижний.
Плитки в ряду — слева направо. Координата плитки в режиме
укладки: «Стена N, ряд K (от пола), плитка M слева».

Сквозная нумерация плиток в ряду идёт без учёта масок —
плитка 12 в ряду 5 это всегда 12-я плитка слева, даже если
в этой позиции дыра-маска.

### Остаток

Если стена не делится нацело — крайняя плитка обрезается
физически до нужного размера. В спецификации она идёт как
целая плитка (либо с пометкой «обрезная» в режиме укладки).
Растяжка швов на краях из v1 убрана как несовместимая с масками.

### Учёт масок при расчёте количества

После расчёта сетки на стене из неё вычитаются плитки, попавшие
**целиком** в маски-препятствия. Спецификация показывает только
реально нужные плитки.

## Лимиты на количество плиток

При расчёте сетки приложение проверяет общее число плиток на стену
(до вычета масок):

- **Мягкий лимит:** 25 000 плиток на стену. При превышении —
  предупреждение в форме настройки плитки: «На стене N будет
  X плиток. На мобильных устройствах может тормозить».
- **Жёсткий лимит:** 75 000 плиток на стену. При превышении —
  блокировка с требованием увеличить размер плитки или уменьшить
  стену. Применение настроек запрещено.

Лимиты обусловлены производительностью InstancedMesh на мобильных
устройствах.
