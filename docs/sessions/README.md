# Сессионные промпты — проект Drape

## Как использовать

Каждая сессия:
1. Читает `state.md` — результаты предыдущих сессий
2. Выполняет свою работу
3. Дописывает свой блок в `state.md` перед завершением

Файл `state.md` — живой документ, накапливающий все решения.

## Правило приоритета промптов

**Файлы `prompts/*.md` — главный и единственный авторитетный источник задач.**

Блок «Контекст для следующей сессии X.X» в `state.md` — это только дополнительный контекст (что сделано, что осталось). Он **не может** перенаправлять задачи или заменять заготовленный промпт из `prompts/`.

Если handoff в `state.md` противоречит файлу `prompts/X.X-*.md` — всегда следовать файлу из `prompts/`. Handoff в `state.md` — лишь примечание, не команда.
