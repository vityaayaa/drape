## Архитектурное понимание (прочитай внимательно)

### Данные для режима укладки

Все данные уже есть в store, новых полей не нужно:

```
walls[]                    — стены с параметрами и масками
tile                       — глобальные параметры плитки
pixelizer.tileColors       — wallId → { 'col_row': '#rrggbb' }
```

**Ключевой факт о `tileColors`:** ключ — строка `'col_row'` (например `'5_3'` = колонка 5, ряд 3). Ряд 0 — ВЕРХНИЙ (как в canvas), ряд (rows-1) — нижний. Но в режиме укладки пользователю показывается **ряд от пола** — физический порядок укладки. Конвертация: `rowFromFloor = totalRows - 1 - canvasRow`.

**Floor anchor:** плитки выровнены по нижнему краю стены. Первый полный ряд — снизу. Неполный ряд сверху — обрезанные плитки. Это реализовано в schemaRenderer через `buildSchemaLayout` — используй ту же логику.

### Маски

Функция `isFullyInsideMask` из `pixelizerGeometry.js` принимает параметр `tileStartY_mm` — смещение сетки от floor anchor. Плитки, полностью попавшие в маску, **пропускаются** при обходе.

### Расчёт сетки

```js
resolveWallTile(wall, globalTile) // из schemaRenderer.js — объединяет tile_overrides с globalTile
// потом:
cols = floor((width_mm + groutW) / (tileW + groutW))
rows = floor((height_mm + groutW) / (tileH + groutW))
// width_mm = parseFloat(wall.length) * 10  (см → мм)
// height_mm = parseFloat(wall.height) * 10
```

### Режимы обхода

**По рядам (default):** stены по порядку. Внутри стены: снизу вверх (ряды от пола), в каждом ряду слева направо. Маски пропускаются. Переход между стенами автоматический.

**По цветам:** сначала все плитки цвета №1 (все стены), потом все плитки цвета №2, и т.д. Внутри одного цвета: сначала все плитки стены 1, потом стены 2, и т.д. Внутри стены — в том же порядке (ряды снизу вверх, слева направо). Маски пропускаются.

### Плитка без цвета

Если для плитки нет записи в `tileColors` (пикселизация не запущена) — цвет плитки `grout_color` (или `#888888` как fallback). Такие плитки всё равно включаются в обход — мастер должен видеть весь план укладки, даже без цветов.

## СЕССИЯ 4.1 — Дизайн: экран укладки

**Скилл: `/ui-ux-pro-max`**

Прежде чем принимать решения — детально разберись в контексте использования. Это приложение открывают прямо во время укладки плитки: руки могут быть в клее, экран смотрят мельком. Каждый элемент должен быть крупным, контрастным, читабельным с расстояния 40 см.

### Что нужно продумать и решить:

**1. Общая компоновка экрана**

Предложи 3 варианта компоновки: как расположены блок текущей плитки, мини-превью стены и кнопки навигации. Для каждого — плюсы и минусы. Затем выбери лучший и обоснуй.

Учти: экран телефона ~390×844px, нижний таббар занимает ~57px. Доступная высота ≈ 787px.

Ключевой вопрос: как уместить всё без скролла? Если не получается — что скрыть/свернуть?

**2. Блок текущей плитки**

Это главный элемент. Продумай:
- Квадрат цвета — какого размера? Где расположен?
- Координаты «Стена N, ряд K от пола, плитка M слева» — как форматировать? Всё в одну строку или разбить?
- Номер цвета из палитры — где?
- Размер плитки в мм — нужен ли всегда или только по запросу?
- Hex-код цвета — показывать или нет?
- Счётчик прогресса «247 из 1820» — где живёт?

**3. Мини-превью стены**

Это Canvas-элемент, рисующий сетку плиток текущей стены. Нужно думать о:
- Размер превью на экране (высота × ширина)
- Подсветка текущей плитки — цвет подсветки (не совпадает с цветом плитки), толщина обводки
- Подсветка нескольких соседних плиток — сколько соседних и как именно (приглушенные цвета? полупрозрачный оверлей?)
- Тап на плитку → переход к ней — как визуально отобразить что плитка кликабельна?
- Прокрутка: стена может быть очень широкой. Как скролл? Авто-центрирование на текущей плитке?
- Уже уложенные плитки — отмечать как-то? (checkbox? затемнение? checkmark?)

**4. Навигация**

- Кнопки «Предыдущая» / «Следующая» — размер, расположение, жест swipe?
- Кнопка «К плитке…» — что это такое? Диалог? Ввод номера? Или выбор из превью?
- Переключатель режима (по рядам / по цветам) — где? Всегда видна или убрана за меню?

**5. Смена стены**

В режиме «по рядам» после последней плитки стены 1 переходим на стену 2.
- Как уведомить пользователя о переходе? Toast? Модалка? Автоматически?
- Показывать ли название следующей стены заранее?

**6. Завершение**

После последней плитки всех стен — что показать? Экран «Готово»? Конфетти? Возврат в начало?

**7. Пустые состояния**

Два случая:
- Нет стен вообще → EmptyState с кнопкой → Комната
- Стены есть, но пикселизация не запущена → можно работать (плитки без цвета), но нужно предупреждение

**5. Доступность при физическом использовании**

**6. Анимации переходов**

- Переход к следующей плитке — анимировать блок текущей плитки? (slide? fade? scale?)

**7. Toolbar / заголовок вкладки**

- Кнопка «выйти из режима укладки» — куда? В «Схему»? В «Фото»?
