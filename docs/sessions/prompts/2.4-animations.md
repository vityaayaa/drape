## Промпт

Прочитай `docs/sessions/state.md` — там дизайн-решения из фазы 1 (особенно 1.1 и 1.5 про анимации и переходы) и результаты предыдущих кодинг-сессий.

Добавляем анимации в **Drape**. Стек: React 18. Рассмотреть Framer Motion (уже может быть в зависимостях) или CSS transitions.

**Прочитай `package.json`** — проверь установлен ли framer-motion. Если нет — обсудим нужно ли ставить.

**Задачи:**

**З1 — Добавление/удаление стены**
- Добавление: новая WallCard появляется с animation (slide down + fade in)
- Удаление: WallCard исчезает с animation (slide up + fade out) перед удалением из store

**З2 — Добавление/удаление маски**
Аналогично: MaskCard появляется/исчезает с анимацией.

**З3 — Переключение вкладок**
Реализовать transition из сессии 1.5 (fade / slide — по выбору дизайна).

**З4 — Feedback на нажатие кнопок**
- Кнопка "+ Добавить стену": brief scale или ripple при tap
- Кнопка "Пикселизировать": loading state с анимацией пока идёт обработка (уже есть `sampling` state)
- Кнопка удаления: brief red flash

**З5 — Bottom sheets**
WallSelectSheet и WallsSheet: появление снизу (slide up), исчезновение (slide down).

**З6 — Toast уведомление**
Уже есть Toast компонент — добавить slide in/out анимацию.

**З7 — Индикатор автосохранения**
Реализовать из решения сессии 1.5: показывать "Сохранено ✓" после изменений с fade in/out.

**Принципы:**
- Анимации не должны мешать работе — быстрые (150-300ms), не блокирующие
- На слабых устройствах учесть `prefers-reduced-motion`
- Не анимировать то, что пользователь делает часто (будет раздражать)
