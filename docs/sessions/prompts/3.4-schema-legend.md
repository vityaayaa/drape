## Промпт

Прочитай `docs/sessions/state.md` — там контекст проекта, дизайн-решения сессий 3.1–3.2 и результат сессии 3.3 (рендер развёртки, `buildPalette`).

Мы реализуем нижнюю часть split-screen вкладки **«Схема»**: легенду цветов и спецификацию.

**Прочитай перед началом:**
- `src/components/export/ExportTab.jsx` — текущее состояние после сессии 3.3
- `src/components/export/SchemaView.jsx`
- `src/utils/schemaRenderer.js` (особенно `buildPalette`)
- `src/store/projectStore.js` — структура `tile`, `walls`

**Задачи:**

**З1 — Компонент `SchemaLegend`**
Новый файл `src/components/export/SchemaLegend.jsx`. Нижняя прокручиваемая часть split-screen.

Шапка (фиксированная, не скроллится):
- Размер плитки (ширина × высота мм)
- Ширина шва, цвет шва (квадратик + мм)
- Поле «Запас, %» — стиль и расположение из сессии 3.2

Список цветов (скроллится):
- Строка цвета: квадратик (16×16, borderRadius 3) + «#» + порядковый номер + hex-код + «N шт.» + при запасе > 0: «→ M шт.»
- Разбивка по стенам — стиль из сессии 3.2 (раскрываемый или сразу видный)
- Пустое состояние: если palette пуста — hint «Запустите пикселизацию во вкладке Фото»

**З2 — Логика запаса**
Функция `withSurplus(count, surplusPercent): number` — `Math.ceil(count * (1 + surplusPercent / 100))`.
`surplusPercent` хранится в локальном state компонента (не в store — это параметр печати, не проекта).
При изменении поля «Запас %» — пересчитать все строки легенды.

**З3 — Интеграция в `ExportTab`**
Заменить placeholder нижней части на `<SchemaLegend palette={palette} tile={tile} walls={walls} />`.
`palette` — результат `buildPalette` из сессии 3.3 — передавать из `ExportTab` в оба компонента (единый источник).
