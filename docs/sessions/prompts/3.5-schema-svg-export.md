**Задачи:**

**З1 — Генератор `buildSchemaSVG`**
Новый файл `src/utils/schemaSVGBuilder.js`.

Принимает `{ walls, tile, tileColors, palette, options }`, где `options`:
- `scale: 'fit' | 'real'` — «На экран» или «Реальный размер 1:1»
- `drawGrout: boolean` — рисовать швы
- `surplusPercent: number` — для легенды в SVG

Возвращает строку SVG.

Содержимое SVG:
- Развёртка стен (все стены горизонтально, плитки с цветами, маски, названия стен)
- Легенда под развёрткой: квадратик цвета + номер + hex + количество + с запасом
- При `scale: 'real'` — атрибуты `width`/`height` в мм (`width="1200mm"`)
- При `scale: 'fit'` — `viewBox` с удобными единицами, `width="100%"`
- При большом числе плиток (> 500 уникальных цветов или > 10 000 плиток) использовать `<symbol>`/`<use>` для оптимизации: каждый цвет определить один раз как `<symbol>`, переиспользовать `<use>`

**З2 — Диалог настроек экспорта**
Новый файл `src/components/export/ExportDialog.jsx`. Стиль — bottom sheet (как `WallsSheet` в Фото).

Элементы (дизайн из сессии 3.2):
- Заголовок «Настройки экспорта»
- Два radio-варианта масштаба: «На экран» (default) / «Реальный размер 1:1»
- Toggle «Рисовать швы» (disabled и выключен при `grout_width = 0`)
- Кнопка «Скачать SVG» (primary) — при нажатии генерирует SVG и вызывает `downloadFile`

Функция `downloadFile(svgString, filename)` — создаёт `Blob`, временную ссылку, триггерит скачивание.

Состояние загрузки (из сессии 3.2): пока генерируется SVG — спиннер на кнопке, кнопка disabled.

**З3 — Интеграция кнопки в `ExportTab`**
Кнопка «Скачать SVG» в toolbar (расположение из сессии 3.1) — открывает `ExportDialog`.
Передать в диалог `palette`, `surplusPercent` (из `SchemaLegend`), текущие `walls`, `tile`, `tileColors`.

**Важно:**
- Генерация SVG может занять несколько секунд для большой комнаты — запускать в `setTimeout(0)` чтобы не блокировать UI перед показом спиннера
- SVG генерируется только в браузере, без сервера — никаких fetch-запросов
