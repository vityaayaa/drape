## Промпт

Прочитай `docs/sessions/state.md` — там все дизайн-решения фазы 1 (особенно 1.5 — онбординг) и результаты всех предыдущих кодинг-сессий.

Финальная кодинг-сессия — реализация **онбординга и пустых состояний** во всех вкладках.

**Прочитай перед началом:**
- `src/App.jsx` — навбар, переключение вкладок
- `src/components/room/RoomTab.jsx`
- `src/components/viewer/ViewerTab.jsx`
- `src/components/export/ExportTab.jsx`
- `src/components/layout/LayoutTab.jsx`
- `docs/sessions/state.md` раздел сессии 1.5 — конкретные решения по каждому пустому состоянию

**Задачи:**

**З1 — Пустое состояние Комнаты**
Реализовать решение из сессии 1.5: что показывать новому пользователю до добавления первой стены.

**З2 — Пустое состояние 3D**
Сейчас: "Добавьте хотя бы одну стену с размерами на вкладке «Комната»" — простой текст.
Реализовать дизайн из сессии 1.5.

**З3 — Заглушки Схемы и Укладки**
Реализовать красивые заглушки из сессии 1.5 (tease-карточки / wireframe preview / другое).

**З4 — Прогресс-индикатор**
Если в сессии 1.5 решено добавить — реализовать. Если нет — пропустить.

**З5 — Индикатор автосохранения**
Подключиться к Zustand store: при каждом изменении данных (after persistence.js сохранит) показать индикатор из сессии 1.5.
Посмотри `src/store/persistence.js` — как сейчас работает авто-сохранение.

**З6 — Связи между вкладками**
Убедиться что все кнопки "Перейти в Комнату →" в пустых состояниях работают через `setActiveTab`.
Добавить аналогичные навигационные подсказки где их нет.
