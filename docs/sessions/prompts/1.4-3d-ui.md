## Промпт

Прочитай файл `docs/sessions/state.md` — там контекст проекта, список проблем и результаты предыдущих сессий.

Ты дизайнер мобильного PWA **Drape**. Сегодня работаем над вкладкой **«3D»**.

**Прочитай эти файлы:**
- `src/components/viewer/ViewerTab.jsx` — Canvas + OrbitControls, позиция камеры
- `src/components/viewer/RoomScene.jsx` — сцена: стены
- `src/components/viewer/WallMesh.jsx` — один меш стены (PlaneGeometry + текстура плитки)

**Текущее состояние 3D:**
- Three.js / React Three Fiber
- OrbitControls (вращение orbit + zoom + pan)
- frameloop="demand" (рендер по требованию — важно для батареи)
- Камера смотрит снаружи комнаты (нужно изнутри или isometric)
- PlaneGeometry — стены без толщины
- Освещение: ambientLight + directionalLight снаружи
- Нет пола, нет UI-оверлеев, нет кнопок управления видом

**Эталон для вдохновения:** Sketchfab viewer (sketchfab.com) — удобный orbit, красивое освещение, минималистичный UI, не лагает.

**Задачи:**

1. **Камера по умолчанию**: какой вид лучше для интерьерного просмотра комнаты с плиткой?
   - Изнутри (POV пользователя, стоящего в центре)
   - Изометрия сверху-сбоку (видна вся комната)
   - Orbiting вокруг центра снаружи (текущий, но с fit)
   Покажи плюсы/минусы каждого. Я выберу.

2. **Сетка пола**: нужна reference-сетка на нулевой отметке (как в Blender). Какой стиль? (цвет, прозрачность, шаг сетки)

3. **Толщина стен**: переход с PlaneGeometry на BoxGeometry. Какая визуальная толщина правильная? (реалистично ~100-150мм для стены с плиткой)

4. **Освещение**: для интерьерного просмотра нужен другой сет. Предложи: тип источников (ambient + точечный в центре? HDR environment?), интенсивность, цвет.

5. **UI-оверлеи**: что нужно поверх 3D-канваса?
   - Кнопка "Сбросить камеру"
   - Переключатели вида (спереди / сверху / изометрия)
   - Подсказка управления при первом открытии
   - Названия стен (labels)?
   Покажи предлагаемое расположение элементов на схеме экрана.

6. **Управление жестами на mobile**: OrbitControls работает, но может быть неинтуитивен. Нужно ли:
   - Ограничить угол наклона (не смотреть снизу вверх)?
   - Добавить damping (инерцию)?
   - Ограничить zoom?

7. **Стиль UI-оверлеев**: в стиле дизайн-системы из сессии 1.1 — glass-кнопки? flat? pill-shaped?
