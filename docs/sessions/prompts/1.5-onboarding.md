## Промпт

Прочитай файл `docs/sessions/state.md` — там полный контекст проекта и результаты всех предыдущих дизайн-сессий.

Ты дизайнер мобильного PWA **Drape**. Последняя дизайн-сессия — **онбординг и визуальная связь между вкладками**.

**Текущее состояние:**
- Новый пользователь открывает приложение — видит пустую форму Комнаты. Нет объяснения что делать.
- Пустые состояния есть только на Фото ("Нет стен → перейди в Комнату") — это хороший паттерн.
- Вкладки 3D и Фото просто пустые если нет данных, без подсказки.
- Схема ("Раздел появится в этапе 6") и Укладка ("этап 7") — просто тёмный экран с текстом.
- Нет прогресс-индикатора, нет flow-направления.

**Рабочий flow приложения:**
Комната (настройка) → Фото (наложение) → 3D (просмотр) → Схема (экспорт) → Укладка (план)

**Задачи:**

1. **Пустое состояние Комнаты**: что показать новому пользователю вместо голой формы? Короткий onboarding? Анимированная подсказка? Сохрани существующую форму — только добавь контекст.

2. **Пустые состояния Фото и 3D**: уже есть "Нет стен → перейди в Комнату". Как улучшить? Нужна иллюстрация? Кнопка-ссылка на предыдущий шаг?

3. **Заглушки Схемы и Укладки**: сейчас "Раздел появится в этапе N". Можно ли сделать красивее — показать wireframe/preview того что будет? Или tease-карточку?

4. **Визуальный flow**: нужен ли какой-то прогресс-индикатор (1-2-3-4-5 шагов)? Или достаточно умных пустых состояний?

5. **Индикатор автосохранения**: данные пишутся в IndexedDB автоматически, но пользователь об этом не знает. Показать "Сохранено" через N секунд после изменений?

6. **Анимации переходов**: при переключении вкладок — нужен ли transition? Какой? (fade, slide, none)
