## Промпт

Прочитай `docs/sessions/state.md` — там контекст проекта, **дизайн-решения сессии 1.4** (камера, пол, освещение, UI-оверлеи) и результат сессии 2.1.

Мы переделываем вкладку **«3D»**. Стек: React Three Fiber (R3F), @react-three/drei, Three.js.

**Прочитай перед началом:**
- `src/components/viewer/ViewerTab.jsx`
- `src/components/viewer/RoomScene.jsx`
- `src/components/viewer/WallMesh.jsx`
- `src/components/viewer/MaskOverlay.jsx`
- `src/utils/computeWallPositions.js`
- `src/utils/buildTileTexture.js`

**Задачи (в порядке исполнения):**

**З1 — Камера по умолчанию**
Реализовать вид из сессии 1.4. Камера должна автоматически вмещать всю комнату в экран (auto-fit). При изменении набора стен — пересчитывать позицию.

**З2 — Объёмные стены**
Заменить `PlaneGeometry` на `BoxGeometry` с толщиной (значение из сессии 1.4, обычно 100-150мм → пересчитать в units).
Текстуру плитки показывать только на внутренней грани (backSide или отдельный материал для каждой грани).

**З3 — Сетка пола**
Добавить `gridHelper` или кастомный `<Grid>` из @react-three/drei на нулевой отметке. Параметры из сессии 1.4.

**З4 — Освещение изнутри**
Реализовать освещение из сессии 1.4 (ambient + точечный источник в центре комнаты / HDRI environment). Убрать directionalLight снаружи или скорректировать.

**З5 — UI-оверлеи**
Поверх Canvas добавить HTML-оверлей (position: absolute) с элементами из сессии 1.4:
- Кнопка сброса камеры
- Кнопки видов (front/top/iso)
- Подсказка управления (показать один раз при первом открытии, потом скрыть)

**З6 — Управление OrbitControls**
Настроить по результатам сессии 1.4: damping, ограничения угла, minDistance/maxDistance, скорость зума.

**З7 — Пустое состояние**
Реализовать дизайн пустого состояния из сессии 1.5.

**Важно:** frameloop="demand" оставить — это критично для батареи на mobile. После любого изменения камеры или данных вызывать `invalidate()`.
