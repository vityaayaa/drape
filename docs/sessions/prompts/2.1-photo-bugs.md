## Промпт

Прочитай `docs/sessions/state.md` — там контекст проекта и все дизайн-решения из фазы 1.

Мы реализуем исправления вкладки **«Фото»** (PixelizerTab). Это React 18 + Zustand + react-zoom-pan-pinch + Canvas 2D.

**Прочитай перед началом:**
- `src/components/pixelizer/PixelizerTab.jsx`
- `src/components/pixelizer/ControlsPane.jsx` — особенно `TransformPane` и `Field`
- `src/components/pixelizer/PhotoPanorama.jsx` — TransformWrapper настройки
- `src/components/pixelizer/WallCanvas.jsx`
- `src/utils/pixelizerRenderer.js`

**Баги для исправления (по приоритету):**

**П1 — Режим трансформации фото (жесты)**
Сейчас: TransformWrapper в PhotoPanorama перехватывает ВСЕ жесты. Однопальцевый = панорамирование мира, пинч = зум мира. Фото двигается только через числовые поля.
Нужно: в режиме `transform` (uiMode==='transform') — однопальцевый жест двигает активное фото (обновляет offsetX_mm/offsetY_mm), пинч масштабирует активное фото (обновляет scale).
В режиме `navigate` — всё работает как сейчас (мир).
Подсказка: возможно нужно disabled TransformWrapper в transform-режиме и добавить собственные touch handlers на canvas активного фото.

**П2 — Яркость/Контраст/Насыщенность не работают**
Сейчас: значения хранятся в store (photoSettings.brightness/contrast/saturation), но в WallCanvas и pixelizerRenderer они не применяются.
Нужно: применить через `ctx.filter = \`brightness(${b}) contrast(${c}) saturate(${s})\`` перед drawImage.

**П3 — Сломанный ввод числовых полей (scale, offsets)**
Сейчас: `Field` компонент — controlled input с `value={value.toFixed(2)}`. React перезаписывает DOM при каждом keystroke → пользователь не может нормально ввести число.
Нужно: переделать на uncontrolled pattern с onBlur, или использовать локальный state для строки ввода.

**П4 — iOS leading zero ("050")**
Сейчас: при вводе offsetX=50 на iOS поле показывает "050".
Нужно: `inputMode="numeric"` или `type="text"` + валидация вместо `type="number"` для этих полей.

**П5 — Отрицательный сдвиг**
Нужно: убедиться что offsetX_mm и offsetY_mm могут быть отрицательными. Убрать любые min=0 ограничения.

**П6 — Выравнивание плиток по нижнему краю (floor anchor)**
Сейчас: стены разной высоты — плитки начинаются с разной позиции по Y.
Нужно: плитки всегда выровнены снизу вверх (как в реальной комнате). Проверить в `pixelizerRenderer.js` начальную координату Y.

**П7 — Качество изображения**
В WallCanvas при рисовании: убедиться что `ctx.imageSmoothingEnabled = true` и `ctx.imageSmoothingQuality = 'high'`.

**Подход:** Разбери каждый баг, предложи решение, реализуй последовательно. Используй superpowers:test-driven-development где применимо.
