## ШАГ 1 — Code Review через скилл

Запусти `/superpowers:requesting-code-review`.

Задача ревьюеру: проверить весь проект целиком, начиная с первого коммита до HEAD.

```bash
BASE_SHA=$(git log --oneline | tail -1 | awk '{print $1}')
HEAD_SHA=$(git rev-parse HEAD)
```

Что проверить ревьюеру (передай это как PLAN_OR_REQUIREMENTS):
- Корректность алгоритмов пикселизации (pixelizerGeometry, pixelizerRenderer, pixelizerSampler)
- Корректность floor anchor и isFullyInsideMask
- Zustand store: нет ли утечек, лишних ре-рендеров, stale closures
- React паттерны: useEffect dependencies, useMemo/useCallback где нужны
- Canvas рендер: нет ли memory leaks (незакрытые контексты, removeEventListener)
- Доступность (a11y): touch targets ≥ 44px, контраст текста
- Обработка ошибок: что происходит при невалидных данных в store

## ШАГ 3 — Технический аудит (сделай сам)

После скиллов — проведи собственный анализ по каждому разделу.

### 3.1 Производительность

- Запусти `npm run build` и посмотри размер бандла. Есть ли что оптимизировать (code splitting, lazy imports)?
- Проверь: есть ли `React.lazy` / `Suspense` для тяжёлых вкладок (3D = Three.js, это большая библиотека)?
- Есть ли `useMemo` там где canvas перерисовывается при каждом рендере?
- Web Worker для пикселизации — реально ли используется или fallback на main thread?

### 3.2 Тесты

- Запусти `npm test` — все ли зелёные?
- Посмотри покрытие: какие части кода **не** покрыты тестами? Особенно критичные алгоритмы.
- Есть ли тесты для schemaRenderer, schemaSVGBuilder? (Из state.md известно что их не было)
- Есть ли тесты для persistence.js (IndexedDB)?

### 3.3 PWA

- Проверь `vite.config.js` — настроен ли `vite-plugin-pwa`?
- Есть ли `public/manifest.json`? Все ли поля заполнены (name, icons, theme_color, display)?
- Работает ли офлайн? (Service worker precache всех ассетов)
- Проверь: добавлены ли иконки для iOS (apple-touch-icon)?

### 3.4 Доступность и UX

- Все ли кнопки имеют `aria-label` или видимый текст?
- Все ли интерактивные элементы имеют минимальный touch target 44×44px?
- Есть ли `prefers-reduced-motion` — да, есть в animations.css, но охватывает ли все анимации?
- Работает ли приложение при `font-size: 200%` в системных настройках iOS?

### 3.5 Хранилище и данные

- IndexedDB: что происходит если хранилище переполнено (quota exceeded)?
- Что происходит если IndexedDB недоступна (режим инкогнито Safari)?
- Экспорт JSON с base64 фото — есть ли ограничение на размер файла при скачивании?
- При импорте повреждённого файла — проверяется ли каждое поле или может крашнуться?

### 3.6 Мобильная специфика

- Проверь: нет ли `100vh` проблем на iOS (нижняя адресная строка Safari)?
- Safe area insets — применены ли везде где нужно (`env(safe-area-inset-bottom)`)?
- Canvas на Retina: есть ли `devicePixelRatio` учёт в WallCanvas и SchemaView?
- Горизонтальная ориентация — как ведёт себя layout?

### 3.7 Безопасность

- При импорте JSON проекта — делается ли sanitize данных перед `restoreSnapshot`?
- Фото в base64 — валидируется ли что это действительно изображение?
- Нет ли `dangerouslySetInnerHTML` или `eval`?
