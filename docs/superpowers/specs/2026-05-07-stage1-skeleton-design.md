# Этап 1 — Каркас: дизайн

**Дата:** 2026-05-07  
**Ветка:** `stage_1`  
**Этап по ТЗ:** Этап 1 — Каркас (TECH_STACK.md)

---

## Цель этапа

Создать фундамент приложения: рабочий PWA с навигацией, 3D-движком,
автосохранением и историей действий. Без бизнес-функций — только
инфраструктура, на которую навешивается всё последующее.

По завершении этапа в браузере (и на телефоне через Vercel) есть:
- Установленный PWA (добавляется на главный экран)
- 5 вкладок, переключаются, состояние каждой сохраняется
- Вкладка «3D» с вращающимся кубом (проверка R3F)
- Остальные 4 вкладки — заглушки с кнопками undo/redo
- Автосохранение в IndexedDB (восстановление при перезагрузке)
- Автодеплой на Vercel при каждом пуше

---

## Навигация

**Нижний таббар — 5 вкладок:**

```
┌─────────────────────────────┐
│                             │
│      [контент вкладки]      │
│                             │
│   (undo/redo внутри         │
│    каждого раздела)         │
├─────────────────────────────┤
│ Комната │ Фото │ 3D │ Схема │ Укладка │
└─────────────────────────────┘
```

- Нет глобальной шапки с названием приложения — лишняя высота на мобильном
- Все 5 вкладок видны сразу и кликабельны; реальный контент только в «3D»
- Неактивные вкладки (будущие этапы) — визуально приглушены (серые иконки)
- Нажатие на неактивную вкладку — разрешено, показывает заглушку
- При переключении вкладок их UI-состояние сохраняется (все вкладки
  живут в памяти одновременно, видна только активная)
- Кнопки undo/redo размещаются внутри каждой вкладки (точное место
  определится в следующих этапах; на этапе 1 — просто кнопки в заглушке)

**Активная вкладка по умолчанию:** «3D» — единственная с реальным содержимым на этапе 1; остальные открываются, но показывают заглушку

---

## Что на экране

**Вкладка «3D»:**
- Вращающийся куб через React Three Fiber
- Занимает всю доступную высоту над нижней панелью
- Кнопки ↩ ↪ (undo/redo) где-то в углу

**Вкладки «Комната», «Фото», «Схема», «Укладка»:**
- Серый экран, название раздела по центру
- Кнопки ↩ ↪ (undo/redo)
- Состояние (в будущем: скролл, открытые панели) сохраняется

## IndexedDB — схема с первого дня

Две таблицы:

- **`project`** — настройки (комната, плитка, маски, параметры палитры,
  кадрирование, активная вкладка). Перезаписывается при каждом изменении.
- **`photos`** — фотографии по id. Хранятся отдельно, не попадают в историю.

На этапе 1 в `project` хранится только `activeTab` и тестовое состояние
для проверки работы IndexedDB.

---

## Undo/redo — скелет

- Стек снимков состояния в `historyStore` (Zustand)
- Снимок = всё состояние `projectStore` без фотографий
- Лимит: 50 снимков (старые вытесняются)
- Снимки персистируются в IndexedDB (восстанавливаются при перезагрузке)
- На этапе 1: кнопки ↩ ↪ меняют тестовое состояние (счётчик или цвет куба)

## PWA

- `manifest.json`: название «Drape», иконка, `display: standalone`,
  `theme_color`, `background_color`
- Service worker через `vite-plugin-pwa`: precache всех ассетов сборки
- После первой загрузки приложение работает офлайн
- На телефоне браузер предлагает «Добавить на главный экран»

---

## Vercel

- GitHub-репозиторий подключается к Vercel один раз (через веб-интерфейс)
- Каждый `git push` в любую ветку → автоматический preview-деплой
- Ветка `stage_1` получает постоянную preview-ссылку для тестирования
  на телефоне
