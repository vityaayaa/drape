## 1. Исправление логики (`src/utils/staircase.js`)

### 1.1 `calcStaircase` — перепутаны режимы

Условие treadsCount перевёрнуто — поменять `standard` → `immediate`:

```js
// ДО (ошибка):
const treadsCount = startType === 'standard' ? Math.max(1, N - 1) : N

// ПОСЛЕ:
const treadsCount = startType === 'immediate' ? Math.max(1, N - 1) : N
```

Результат:
- `immediate`: treadsCount = N-1 → treadDepth = L/(N-1) → **БОЛЬШЕ** ✓
- `standard`: treadsCount = N → treadDepth = L/N → **МЕНЬШЕ** ✓

### 1.2 `calcStaircase` — формула угла

Угол должен отражать эргономику шага, а не суммарный наклон:

```js
// ДО:
const angle = Math.atan(H / L) * (180 / Math.PI)

// ПОСЛЕ:
const angle = Math.atan(riserHeight / treadDepth) * (180 / Math.PI)
```

Результат: `immediate` угол < `standard` угол при одинаковых H и L ✓

### 1.3 `buildStaircaseMasks` — двойная высота и выход за пределы

При `immediate` с N-1 проступями последняя маска вылезает за пределы стены. Фикс:

```js
// ДО:
const stepsTotal = N

// ПОСЛЕ:
const stepsTotal = startType === 'immediate' ? N - 1 : N
```

Также удалить первую маску у `standard` (h=0 уже пропускается через `if (h <= 0) continue`).

### 1.4 `buildStaircaseMasks` — добавить Y-сдвиг

```js
// Параметр: startY = 0 (по умолчанию)
// В маске: y: round2(startY)  (вместо y: 0)
```

Семантика X/Y:
- **X** = от левого края стены до точки входа (нижней ступени)
  - `→ Вправо`: X = левый край нижней ступени
  - `← Влево`: X = правый край нижней ступени (маски расходятся влево)
- **Y** = от пола до основания лестницы (обычно 0)

## 3. Store (`src/store/projectStore.js`)

### 3.1 Структура wall

```js
wall = {
  id, name, length, height, wall_active, mosaic_active,
  tile_overrides: {},
  masks: [],    // все маски (ручные + лестничные, помечены staircaseId)
  stairs: [],   // конфиги лестниц
}
```

Маска от лестницы: `{ id, name, staircaseId, x, y, width, height, color }`

Стейт лестницы:
```js
{
  id: 'stair-xxx',
  name: 'Лестница 1',
  totalHeight: '150',
  totalLength: '170',
  risers: '7',
  startType: 'immediate',  // 'immediate' | 'standard'
  direction: 'right',      // 'right' | 'left'
  startX: '0',
  startY: '0',
  color: '#888888',
}
```

### 3.2 Новые экшены

- `addStaircase(wallId, config)` — добавить в `stairs[]`, сгенерировать маски с `staircaseId`, добавить в `masks[]`
- `updateStaircase(wallId, stairId, field, value)` — обновить поле в `stairs[]`
- `regenerateStairMasks(wallId, stairId)` — удалить из `masks[]` все с этим `staircaseId`, пересчитать и вставить новые
- `removeStaircase(wallId, stairId)` — удалить из `stairs[]` + удалить все маски с этим `staircaseId`
- `renameStaircase(wallId, stairId, name)` — переименовать

## 4. WallCard (`src/components/room/WallCard.jsx`)

### 4.1 Убрать заголовок «Маски-препятствия»

Заменить на два полноширинных аккордеон-тоггла в одной строке:

```
[🪜 Лестницы ▾]  [≡ Маски ▾]
```

- Лестницы: кастомный SVG-значок ступеней
- Маски: иконка `Fence` из lucide-react

Каждая кнопка переключает свою секцию (открыта/закрыта независимо).

### 4.2 Секция «Лестницы»

```
▼ [🪜 Лестницы]
  [+ Добавить]           ← открывает StaircaseModal (create)
  [Лестница 1] [✏️][🗑]   ← кнопка открывает StaircaseModal (edit)
  [Лестница 2] [✏️][🗑]
```

- Имя редактируется по клику на карандаш (inline input)
- «🗑» удаляет лестницу + все её маски (с подтверждением)
- Клик на «Лестница N» или карандаш → открывает StaircaseModal в режиме edit
- Показывать под именем строку метрик: `21.4 см / 28.3 см / 37° / Идеально`

### 4.3 Секция «Маски»

Показывает только маски без `staircaseId` (ручные).

```
▼ [≡ Маски]
  [+ Добавить]
  MaskCard...
  MaskCard...
```
